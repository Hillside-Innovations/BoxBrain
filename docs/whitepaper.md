# Project Working Title: BoxBrain

## Core Idea

Create a system that remembers where physical objects are, so a person can later search for an item and immediately know which box, bin, shelf, or room it is in. This is not a traditional inventory app. It is a **"memory system for the physical world."**

---

## The Problem

People don't actually lose possessions. They lose location memory.

Moves, renovations, storage units, garages, and job sites all break a person's mental map of where things live. The result:

- duplicate purchases
- hours spent searching
- unfinished projects waiting on a small missing part
- permanent "mystery boxes"

Current inventory tools fail because they require manual typing and organization. During a move or project, nobody has the time or discipline to catalog items one-by-one.

---

## The Insight

Instead of asking the user to describe what is inside a box, we let a camera observe it.

Modern multimodal AI can interpret cluttered scenes and recognize objects (tools, parts, paperwork, hardware, materials, etc.). If we combine:

- a numbered container
- a short video scan
- AI object detection
- semantic search

we can reconstruct a person's spatial memory externally.

**The user does not inventory items.** They simply film the box before closing it.

Later they can search: "allen key" "passport" "sink drain piece" "Makita battery"

**System response:** Box 27 — garage shelf

---

## What Makes This Different

- **Traditional inventory systems** = cataloging possessions  
- **This system** = remembering locations

We are indexing where objects live, not tracking ownership, value, or categories.

The user interacts with it like a search engine: **"Where is ___?"**

---

## Initial Use Case (MVP)

Moving and packing boxes.

**Workflow:**

1. User can label a box. Example use case: "attic_underscore_1"
2. Record 5–10 second video of open box
3. AI detects items
4. Items become searchable
5. Search returns the box number

No manual labeling required.

---

## Why This Matters

The real cost of storage isn't space — it's search time.

Contractors, homeowners, and long-term projects constantly lose time locating parts they already own. A reliable physical memory system reduces:

- duplicate buying
- project delays
- frustration
- clutter accumulation

---

## Potential Future Expansion

After moving boxes, the same system applies to:

- garages and workshops
- construction sites
- tool libraries
- insurance documentation
- estate cleanouts
- warehouses
- shared households

Ultimately this becomes a persistent spatial memory layer for the real world.

---

## Success Criteria (Early)

The project succeeds if a user:

- Packs boxes normally (no special effort)
- Searches for an object days later
- Finds the correct box in under 10 seconds

If that works consistently, the product has real value.
